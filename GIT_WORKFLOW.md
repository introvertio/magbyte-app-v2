# Git Workflow — MagByte App

Remote: https://github.com/magbytex/magbyte-app-v2  
Branch: `main` (commit directly — no feature branches while sole developer)

---

## How to commit and push

### 1. Check what's changed
```bash
git status
git diff
```

### 2. Stage files explicitly — NEVER use `git add -A` or `git add .`
The repo has generated files that must never be committed:
- `next-env.d.ts` — auto-generated by Next.js
- `tsconfig.tsbuildinfo` — TypeScript incremental build cache
- `.claude/worktrees/` — Claude Code internal tool state

Stage only the files you actually changed, e.g.:
```bash
git add CLAUDE.md HANDOVER.md
git add app/dashboard/forecast/page.tsx
git add app/components/ui/dashboard/ChartUtils.tsx
git add app/types/intermediateAnalysis.ts
```

To stage a whole folder safely:
```bash
git add app/
```
`app/` contains no generated files so this is safe. Avoid adding `lib/`, root files, or dotfiles in bulk without checking first.

### 3. Commit
```bash
git commit -m "your message here"
```
Message style: imperative mood, short — "Fix forecast tooltip order" not "Fixed the tooltip".

### 4. Push
```bash
git push origin main
```

---

## Files to never commit

| File / path | Why |
|---|---|
| `next-env.d.ts` | Auto-generated by Next.js on every dev start |
| `tsconfig.tsbuildinfo` | TypeScript build cache — changes every run |
| `.claude/worktrees/` | Claude Code internal — not part of the app |
| `.env` / `.env.*` | Secrets — already in `.gitignore` |

## Quick reference for common sessions

```bash
# See exactly what will be committed before staging
git diff --stat

# Stage all app source changes (safe — no generated files under app/)
git add app/

# Stage root config/docs explicitly
git add CLAUDE.md HANDOVER.md GIT_WORKFLOW.md

# Stage a specific type file
git add app/types/intermediateAnalysis.ts app/types/advancedAnalysis.ts

# Commit and push in one step
git commit -m "your message" && git push origin main
```

## In Cursor

Cursor's built-in Source Control panel uses `git add` under the hood.  
**Do not use "Stage All Changes" (the + next to "Changes")** — it is equivalent to `git add -A` and will include the generated files above.

Instead:
1. Open **Source Control** panel (Ctrl+Shift+G / Cmd+Shift+G)
2. Hover over individual files or folders → click the **+** next to each one to stage them selectively
3. Type your commit message in the box at the top
4. Click **Commit**, then **Sync** (or run `git push origin main` in the terminal)
